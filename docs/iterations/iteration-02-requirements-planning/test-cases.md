# 迭代 2 测试用例

## 1. 测试范围

- 需求创建
- 需求更新
- 需求版本追加
- 计划草稿生成
- 计划确认
- 前端需求表单和计划展示

## 2. 不测试范围

- 真实 Agent 调用
- 多项目数据隔离
- 数据库持久化

## 3. 功能测试用例

### 用例 1：创建需求

- 操作：创建一个完整需求
- 预期：需求创建成功，自动存在版本 `1`

### 用例 2：追加需求版本

- 操作：给已有需求追加版本
- 预期：版本号递增，当前版本切换到最新版本

### 用例 3：生成计划草稿

- 操作：对需求执行生成计划
- 预期：返回 `draft` 状态计划，包含 3 轮迭代和每轮工作包

### 用例 4：确认计划

- 操作：确认某个草稿
- 预期：该计划变为 `confirmed`

### 用例 5：确认第二个计划

- 操作：同一需求生成两个草稿并确认后者
- 预期：只有后者是 `confirmed`，前者变回 `draft`

### 用例 6：前端提交需求

- 操作：在页面中填写并提交需求
- 预期：需求出现在需求列表

### 用例 7：前端触发计划生成

- 操作：点击“Generate plan”
- 预期：计划列表出现新草稿

## 4. 异常流程用例

- 缺少 `title` 创建需求
- 给不存在的需求追加版本
- 确认不存在的计划

## 5. 边界条件用例

- `constraints` 为空
- `acceptanceCriteria` 为空
- 需求内容很短

## 6. 回归范围

- 健康检查接口
- 工作区和 Provider 接口
- 前端基础渲染

## 7. 自动化测试覆盖目标

- 后端需求和计划服务核心路径全部覆盖
- 前端至少覆盖需求提交和计划生成交互
