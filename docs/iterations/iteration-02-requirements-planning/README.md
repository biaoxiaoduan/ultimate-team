# 迭代 2：需求中心与迭代计划生成

## 目标

将“需求拆解为迭代计划”正式做成平台能力，让用户可以录入需求、管理版本、生成计划草稿并确认正式迭代计划。

## 范围

- Requirement 和 RequirementVersion 模型
- 需求创建、编辑、版本记录
- 需求列表和详情展示
- 迭代计划草稿生成
- 迭代计划列表和详情展示
- 计划确认与状态冻结
- 前端需求中心和迭代计划页面

## 不包含

- Agent CRUD
- 多 Agent 编排
- 真实 Provider 调用
- 正式工作包分派到 Agent

## 前置依赖

- 迭代 1 已完成工程底座与 Provider 配置
- 前后端可本地启动

## 交付结果

- 可运行的需求中心接口和页面
- 可运行的迭代计划生成和确认接口
- 版本记录能力
- 对应自动化测试与测试报告

## 对应 Git Commit Message

- `迭代 2：需求中心与迭代计划生成`
