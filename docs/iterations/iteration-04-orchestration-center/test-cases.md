# 迭代 4 测试用例

## 1. 测试范围

- 基于确认计划创建运行
- 阶段状态初始化
- 启动运行
- 执行当前阶段
- 人工确认后进入下一阶段
- 阶段失败与重试
- 运行暂停与继续
- 前端编排中心主要操作流程

## 2. 不测试范围

- 真实 Provider 远程调用
- Git 提交、CI、发布系统集成
- 服务重启后的数据持久化

## 3. 功能测试用例

### 3.1 创建运行

1. 已确认计划存在且 5 个角色 Agent 都启用时，可以成功创建运行
2. 新创建运行状态为 `draft`
3. 首阶段状态为 `ready`，其余阶段为 `pending`

### 3.2 启动和执行

1. `draft` 运行可启动为 `running`
2. 启动后当前阶段变为 `running`
3. 执行当前阶段后，生成一条任务记录
4. 执行当前阶段后，若存在下一阶段，则生成一条交接件
5. 执行后阶段状态变为 `waiting_confirmation`

### 3.3 确认推进

1. 确认当前阶段后，该阶段变为 `completed`
2. 下一阶段变为 `ready`
3. 最后阶段确认后，运行整体变为 `completed`

### 3.4 失败与重试

1. 运行中阶段可标记失败并写入失败原因
2. 阶段失败时运行整体状态变为 `failed`
3. 重试后阶段恢复为 `ready`
4. 重试后运行恢复为 `running`

### 3.5 暂停与继续

1. `running` 运行可暂停为 `paused`
2. `paused` 运行可恢复为 `running`

## 4. 异常流程用例

1. 非确认计划创建运行时返回错误
2. 缺少角色 Agent 时创建运行失败
3. 非当前阶段执行、确认、失败、重试时返回错误
4. 未启动运行前执行阶段时返回错误

## 5. 边界条件用例

1. 同一计划可创建多次运行，每次运行状态独立
2. 最后阶段执行后不会生成下一跳交接件
3. 已完成运行不能再次启动

## 6. 回归范围

- 需求创建和计划确认仍可正常使用
- Agent 创建、编辑、停用、删除仍可正常使用
- 原有前端页面数据加载不受影响

## 7. 自动化测试覆盖目标

- 后端服务和控制器覆盖主要状态机分支
- 前端组件测试覆盖运行创建、阶段执行、确认和失败显示
