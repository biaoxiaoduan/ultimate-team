# 迭代 4：编排中心与阶段流转

## 1. 迭代目标

建立单轮迭代的执行编排能力，让确认后的迭代计划可以被多个角色 Agent 按固定阶段顺序执行，并记录阶段状态、交接物、失败原因与人工确认结果。

## 2. 范围说明

本轮包含：

- `OrchestrationRun`、`RunStage`、`AgentTask`、`Handoff` 数据模型
- 基于确认计划创建运行实例
- 固定阶段流转：产品经理 -> 设计师 -> 开发 -> 测试 -> 发布配置
- 手动启动、执行当前阶段、人工确认、失败标记、重试、暂停、继续
- 编排中心页面与运行状态可视化

本轮不包含：

- 自定义工作流设计器
- 真正调用 Codex / Claude Code 执行命令
- 并行阶段与复杂依赖拓扑
- Git 提交、测试报告、发布配置文件的真实落盘

## 3. 前置依赖

- 已存在确认后的迭代计划
- 已配置并启用对应角色的 Agent
- 已具备基础 Provider 配置能力

## 4. 交付结果

- 后端新增编排运行模块与状态流转接口
- 前端新增编排中心
- 测试覆盖运行创建、阶段推进、人工确认、失败重试、暂停继续
- 本轮文档和测试报告沉淀到独立目录
