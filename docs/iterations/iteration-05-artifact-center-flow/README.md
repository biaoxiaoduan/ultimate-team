# 迭代 5：交付物中心与流程化页面重构

## 1. 迭代目标

在保留前四轮能力的基础上，补齐交付物中心、测试报告与发布记录闭环，同时把前端从单页平铺重构为符合产品方案的流程化多页面结构。

## 2. 范围说明

本轮包含：

- `Artifact`、`TestReport`、`BuildRecord` 数据模型与只读接口
- 基于 Run 阶段完成情况生成交付物与验证记录
- 交付物列表页、交付物详情页、测试发布摘要展示
- 工作台、项目概览、需求列表、需求详情、计划工坊、Agent 中心、Run 列表、Run 详情、交付物中心的流程化页面
- 轻量创建弹窗与失败原因弹窗

本轮不包含：

- 多项目完整 CRUD
- 真实 Git 提交采集
- 真实 CI/CD 执行
- 复杂路由守卫与权限体系

## 3. 前置依赖

- 已具备需求、计划、Agent、Run 基础模型
- 已具备固定阶段编排与阶段控制能力
- 产品方案已更新为流程型页面结构

## 4. 交付结果

- 后端新增交付物与测试发布只读能力
- 前端完成流程型多页面重构
- 从计划到 Run 再到交付物的页面跳转链路打通
- 本轮文档、测试用例、测试报告保留在独立目录
